# quality_merge16_mt_xoshiro

Изолированная рабочая папка для проверки качества `mt19937_64` и `xoshiro256++`
в схеме `16` независимых потоков, склеиваемых в один наблюдаемый поток.

Что здесь лежит:

- `reference_testu01_multi_runner.cpp`
- `reference_rng_streamer.cpp`

Это копии исходных harness-файлов из основного `bench/parallel`, взятые как
опорная точка.

- `testu01_merge16_runner.cpp`
- `rng_streamer_merge16.cpp`

Это специализированные версии для двух генераторов:

- `mt19937_64`
- `xoshiro256pp`

Схема инициализации потоков воспроизводит speed-harness:

- `seed_state = master_seed ^ (tid * 0x9E3779B97F4A7C15)`
- дальше для каждого потока используется `splitmix64` как детерминированный
  источник начального состояния.

Режимы склейки:

- `roundrobin` — по одному `uint64` из каждого из `16` потоков по кругу.
- `block` — блоками по `block_words` значений из каждого потока по кругу.

Для оценки межпоточных корреляций рекомендуемый первый режим — `roundrobin`.
Если нужна более "кусочная" модель потребления, можно использовать `block`.
